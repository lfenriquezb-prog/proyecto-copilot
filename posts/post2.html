Post #2 — Representación de Grafos

Inicio

Formas principales en memoria

1) Lista de Adyacencia

Para cada nodo guardamos una lista (vector, linked-list) con sus vecinos. Muy eficiente en espacio para grafos dispersos.

Ejemplo (mismo grafo del Post #1)

A: B, D
B: A, C, D
C: B, D, E
D: A, B, C
E: C

2) Matriz de Adyacencia

Matriz NxN donde la celda (i,j) indica si existe arista entre i y j. Consultas O(1), pero usa O(N^2) espacio.

Matriz (A,B,C,D,E)

   A B C D E
A: 0 1 0 1 0
B: 1 0 1 1 0
C: 0 1 0 1 1
D: 1 1 1 0 0
E: 0 0 1 0 0

Comparación rápida

  • Lista de adyacencia: Mejor para grafos dispersos; iterar vecinos es eficiente.
  • Matriz de adyacencia: Mejor si necesitas chequear arista frecuentemente en O(1) o si grafo denso.

Visual: ambas representaciones

Las dos vistas aplicadas al mismo grafo ayudan a entender diferencias.
